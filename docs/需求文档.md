## 1. 项目概述

### 1.1 背景

用户在浏览外文网页时，常需要阅读整篇文章的译文，并可将译文保存为常用格式以便离线阅读或归档。本插件面向「单篇文章」场景，提供一键全文翻译与译文下载能力，界面以简洁、低干扰为主。

### 1.2 产品定位

一款基于 **DeepSeek API** 提供翻译能力的 Chrome 浏览器扩展，核心能力为：**当前页面的文章完整翻译** + **译文导出（Markdown、PDF）**。不建设翻译历史、不持久化用户已翻译内容。

### 1.3 目标用户

- 需要阅读外文长文、技术文档、博客等内容的普通用户与知识工作者。
- 希望快速获得整篇译文并本地保存，且不希望插件长期留存阅读记录的用户。

### 1.4 设计原则

- **简约**：页面与交互围绕「翻译」与「下载」两项主流程，避免功能堆砌。
- **一次一用**：翻译结果仅在当前会话中使用，关闭页面或离开站点后不保留业务意义上的「历史记录」。
- **文章为中心**：以当前页面可识别的正文/文章内容为翻译对象，而非零散划词翻译（若未来扩展，不在本文档 v1.0 范围内）。

## 2. 目标与范围

### 2.1 产品目标

| 编号 | 目标 |
|------|------|
| G1 | 用户可在当前网页上一键触发整篇文章翻译，并获得可读性良好的译文展示 |
| G2 | 用户可将译文下载为 Markdown 或 PDF 文件 |
| G3 | 翻译质量由 DeepSeek API 承担，产品侧负责内容组织、展示与导出体验 |
| G4 | 界面与流程简单，新用户无需培训即可完成翻译与下载 |

### 2.2 范围说明（In Scope）

- 识别并提取当前页面的**文章内容**（正文主体）。
- 调用 DeepSeek API 完成**全文翻译**（源语言可为自动检测或由用户指定，见功能需求）。
- 在插件界面或页面内展示**完整译文**。
- 支持将译文**下载为 Markdown、PDF** 两种格式。
- 用户配置翻译所需的基础项（如 API 密钥、目标语言等，见第 4 节）。
- 翻译过程中的状态提示（进行中、成功、失败）。

### 2.3 范围外（Out of Scope）— v1.0 明确不做

- **翻译历史记录**：不保存、不展示、不同步用户曾经翻译过的文章列表。
- **云端账号与多端同步**。
- **划词翻译、段落悬浮翻译、双语对照编辑**等高级阅读模式（除非后续版本单独立项）。
- **对网页 DOM 的原地双语替换**（若与「插件内/侧边展示译文」冲突，以「完整译文阅读 + 下载」为主路径，具体展示方式在 UI 需求中约定）。
- **离线翻译**（无网络时无法完成翻译）。
- **非 Chrome 浏览器**（其他 Chromium 衍生浏览器是否兼容，不作为 v1.0 承诺项）。

## 3. 用户场景

### 3.1 主场景：阅读并保存译文

1. 用户打开一篇外文文章页面。
2. 用户打开插件，确认或选择目标语言，点击「翻译」。
3. 系统提取正文并请求 DeepSeek 完成全文翻译。
4. 用户阅读完整译文；满意后选择「下载」，格式选 Markdown 或 PDF，保存到本地。

### 3.2 辅助场景：翻译失败与重试

1. 因网络、API 限额、密钥无效等原因翻译失败。
2. 用户看到明确失败原因与「重试」入口。
3. 用户修正配置（如密钥）或稍后重试。

### 3.3 辅助场景：长文等待

1. 文章较长，翻译耗时明显增加。
2. 用户看到进度或「翻译中」状态，知晓系统仍在处理，避免重复点击导致重复请求（产品层需定义防重复策略的行为要求，见 4.1.3）。

## 4. 功能需求

### 4.1 文章内容识别与全文翻译

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-001 | 用户可在**当前激活标签页**上发起「整篇文章翻译」。 | P0 |
| F-002 | 系统应能识别页面中的**正文主体**，尽量排除导航、页脚、广告、评论区（若与正文混杂，以「可读正文优先」为产品原则，允许存在少量噪声的说明）。 | P0 |
| F-003 | 翻译应覆盖识别出的正文**全文**，而非仅标题或首段。 | P0 |
| F-004 | 翻译引擎必须使用 **DeepSeek API**；未配置有效密钥时，不得发起翻译并应提示用户完成配置。 | P0 |
| F-005 | 支持设定**目标语言**（如简体中文、英文等）；源语言支持「自动检测」或用户手动选择，具体选项列表在实现阶段确定，但需满足常见外文网页场景。 | P0 |
| F-006 | 译文应**保持原文结构层次**：标题、段落、列表等层级在译文中可辨认，便于阅读与导出。 | P1 |
| F-007 | 对于极长文章，产品应定义可接受的体验边界（如分段翻译、总时长提示）；若单次无法完成，应向用户说明原因而非静默失败。 | P1 |

#### 4.1.1 翻译触发与状态

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-010 | 提供明确的**一键翻译**入口（主按钮文案建议为「翻译」或「翻译全文」）。 | P0 |
| F-011 | 翻译进行中：按钮或界面进入**不可重复提交**状态，并展示「翻译中」类提示。 | P0 |
| F-012 | 翻译成功：展示完整译文内容区域。 | P0 |
| F-013 | 翻译失败：展示**用户可理解的错误说明**（如密钥无效、网络错误、配额不足、内容无法识别等），并提供**重试**能力。 | P0 |

#### 4.1.2 译文展示

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-020 | 译文在插件提供的界面中**完整可读**（支持滚动浏览长文）。 | P0 |
| F-021 | 展示信息宜包含：原文标题（若有）、目标语言、翻译完成时间（可选，仅当前会话展示，**不写入历史库**）。 | P2 |
| F-022 | 不要求在原文网页上逐段对照显示；以「译文阅读区」为主。 | P0 |

### 4.2 译文下载

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-030 | 翻译成功后，用户可将当前译文**下载到本地**。 | P0 |
| F-031 | 支持下载格式：**Markdown（.md）**、**PDF（.pdf）**。 | P0 |
| F-032 | 下载前用户可选择格式；默认可与上次选择一致（**仅限本次浏览器会话内的 UI 记忆**，不构成「翻译历史」）。 | P1 |
| F-033 | 下载文件应包含**完整译文正文**；宜包含文内标题层级（Markdown 用标题语法，PDF 用对应样式）。 | P0 |
| F-034 | 下载文件命名应可读，建议包含：页面标题或站点简称、目标语言、日期（具体命名规则在 UI/交互说明中约定）。 | P1 |
| F-035 | 未翻译成功时，**下载按钮不可用**或隐藏，并避免导出空白文件。 | P0 |
| F-036 | Markdown 导出内容应为纯文本 Markdown，用户可用常见编辑器打开。 | P0 |
| F-037 | PDF 导出应保证中文及常用标点显示正常，版面适合阅读（页边距、字号等以可读为准，不做出版级排版要求）。 | P1 |

### 4.3 配置与安全（产品层）

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-040 | 用户需能配置 **DeepSeek API Key**（必填项），配置入口明显且仅在插件设置/弹出页中维护。 | P0 |
| F-041 | API Key 仅用于本插件调用 DeepSeek，**不得**因「历史记录」功能上传到其他服务端（本版本无历史服务）。 | P0 |
| F-042 | 用户可配置默认**目标语言**等少量选项，选项数量保持精简。 | P1 |
| F-043 | 配置变更后，下一次翻译生效；不要求对进行中任务热更新。 | P2 |

### 4.4 数据与隐私（与「无历史」强相关）

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-050 | **不建立**「已翻译文章」列表、时间线、收藏夹或云端归档。 | P0 |
| F-051 | 不在插件内提供「查看历史上次翻译」入口。 | P0 |
| F-052 | 允许为完成翻译与下载所需的**临时数据**（如当前页提取的正文、当前译文、进行中的任务状态）；用户关闭插件界面、刷新页面或离开站点后，**不应**要求用户仍能找回该次译文（除非用户已下载到本地）。 | P0 |
| F-053 | 若因浏览器机制存在本地缓存（如配置项、会话内 UI 状态），不得将缓存呈现为「翻译历史」功能。 | P0 |
| F-054 | 向用户说明：正文会发送至 DeepSeek 进行翻译，适用 DeepSeek 与用户的相关协议。 | P1 |

## 5. 界面与交互需求

### 5.1 信息架构（简约）

建议仅保留以下模块，避免多余 Tab：

1. **主操作区**：翻译按钮、目标语言（可折叠为简单下拉）。
2. **译文区**：翻译完成后显示全文。
3. **下载区**：格式选择（Markdown / PDF）+ 下载按钮。
4. **设置入口**：API Key、默认语言等（可独立设置页或弹出层，入口单一）。

### 5.2 布局与风格

| 需求 ID | 需求描述 |
|---------|----------|
| UI-001 | 整体风格**简约**，主色与强调色克制，突出主按钮「翻译」与「下载」。 |
| UI-002 | 默认以**弹出窗口（Popup）** 或**侧边面板** 之一承载主流程（实现时二选一，需保证长文可滚动阅读）。 |
| UI-003 | 文案简短，中文为主；错误提示避免错误码堆砌。 |
| UI-004 | 无历史记录相关菜单、列表、搜索框。 |

### 5.3 关键交互流程

```
打开插件 → [未配置 Key] 引导去设置 → 返回
         → 选择目标语言（可选）→ 点击「翻译」
         → 等待（禁用重复点击）→ 阅读译文
         → 选择 Markdown / PDF → 点击「下载」→ 本地保存
```

### 5.4 空态与异常态

| 状态 | 用户可见说明 |
|------|----------------|
| 未配置 API Key | 提示先完成配置，翻译按钮不可用或点击后跳转设置 |
| 页面无正文 | 提示无法识别文章内容，建议换页或稍后重试 |
| 翻译中 | 加载指示，主操作禁用 |
| 翻译失败 | 原因 + 重试 |
| 未翻译 | 下载区不可用；译文区为空态提示 |

## 6. 非功能需求

### 6.1 性能与体验

| 需求 ID | 描述 |
|---------|------|
| NF-001 | 正文提取应在用户可接受时间内完成（目标：数秒内进入「翻译中」状态，具体指标在测试阶段量化）。 |
| NF-002 | 长文翻译允许较长等待，但需有明确进行中反馈。 |
| NF-003 | 译文展示与滚动应流畅，不因一般长度文章出现明显卡顿。 |

### 6.2 可用性

| 需求 ID | 描述 |
|---------|------|
| NF-010 | 核心路径（翻译 → 下载）操作步骤不超过 **3～4 步**（含打开插件）。 |
| NF-011 | 首次使用有简短引导或占位说明（API Key、点击翻译）。 |

### 6.3 兼容性

| 需求 ID | 描述 |
|---------|------|
| NF-020 | 目标平台为 **Google Chrome** 当前主流稳定版本。 |
| NF-021 | 适配常见内容站点（文章页、博客、文档站）；对强反爬或纯 Canvas 页面不承诺 100% 提取成功。 |

### 6.4 安全与合规

| 需求 ID | 描述 |
|---------|------|
| NF-030 | API Key 存储与展示符合浏览器扩展惯例（不明文展示完整密钥、提供修改与清除能力）。 |
| NF-031 | 仅申请完成功能所需的浏览器权限，权限用途可在商店说明中向用户解释。 |

## 7. 内容与格式约定（导出）

### 7.1 Markdown 下载内容

- 文件扩展名：`.md`
- 建议结构示例（逻辑要求，非代码模板）：
  - 一级标题：原文标题的译文（若无标题则使用「译文」+ 日期）
  - 正文：按段落与列表还原为 Markdown 语法
- 编码：UTF-8

### 7.2 PDF 下载内容

- 文件扩展名：`.pdf`
- 包含与 Markdown 一致的**完整正文**
- 可读性优先：正文字号、行距适合屏幕阅读与打印预览，不强制页眉页脚

## 8. 验收标准（Acceptance Criteria）

以下条目可作为 v1.0 是否完成的判定依据：

| 编号 | 验收条件 |
|------|----------|
| AC-01 | 在典型文章页点击「翻译」后，可获得**覆盖正文主体**的完整译文。 |
| AC-02 | 翻译能力依赖 DeepSeek API；错误密钥时无法误用并提示明确。 |
| AC-03 | 翻译成功后，可下载 **Markdown** 文件，用文本编辑器打开内容完整、结构基本清晰。 |
| AC-04 | 翻译成功后，可下载 **PDF** 文件，中文显示正常、内容为完整译文。 |
| AC-05 | 插件界面仅围绕翻译与下载，**无历史记录**相关功能与入口。 |
| AC-06 | 关闭插件或离开页面后，未下载的译文**不提供**历史列表找回（与 F-050～F-052 一致）。 |
| AC-07 | 翻译进行中无法因连续点击产生用户困惑的重复任务（有禁用或合并策略）。 |

## 9. 术语表

| 术语 | 说明 |
|------|------|
| 全文翻译 | 对当前页面识别出的文章正文进行一次性完整翻译 |
| 译文 | DeepSeek API 返回并经产品整理后的目标语言文本 |
| 历史记录 | 用户过往翻译过的文章列表或持久化存档；**本产品不提供** |
| DeepSeek API | 第三方翻译能力提供方，本插件核心翻译依赖 |

## 10. 后续版本（非承诺）

以下能力**不在 v1.0**，仅作 backlog 参考：

- 划词 / 段落翻译
- 原文与译文对照阅读
- 更多导出格式（如 HTML、DOCX）
- 自定义术语表
- 除 DeepSeek 外的模型切换
